As a Lead Prompt Engineer, when crafting a meta-prompt for an advanced model like the Claude Opus tier, we must leverage XML semantic scaffolding, explicit syntactic anchoring, and catastrophic negative constraints.
Advanced Claude models are heavily trained to be "helpful." Left unchecked, this helpfulness manifests as a bias toward standard NPM packages (like nodemailer or bcrypt), TypeScript, and verbose conversational filler. To overwrite these neural weights, we must explicitly map out the only acceptable paths and ruthlessly penalize the unacceptable ones.
Here is the ultimate, bulletproof System Prompt for your App Engine pipeline.

Prompt Engineering Breakdown (Why this works):
XML Scoping & Attention Weights: Claude models are specifically fine-tuned to parse XML architectures. By separating rules into <strict_boundary_constraints>, <architectural_standards>, and <negative_constraints>, the model's attention mechanism logically categorizes constraints rather than jumbling them into a single massive natural-language list.
"Whitelist" vs. "Blacklist" Psychology: It is not enough to say "don't use external packages," because LLMs are probabilistic. If the user asks for email functionality, the weights for nodemailer will heavily activate. By explicitly stating BANNED STANDARD PACKAGES and immediately mapping the functionality to import { Mail } from '@ab/connectors', you cleanly short-circuit and override the model's pre-trained instincts.
Explicit Syntax Skeletons (Few-Shot Injecting): Providing exact code skeletons under <backend_fastify_structure> forces the LLM to adopt your exact syntax style on the very first try, preventing it from inventing its own interpretation of "modular routes."
Behavioral Forcing via Specificity: Catastrophic framing on security constraints is highly effective. Writing "Make it secure" is weak prompting. Explicitly listing the exact failure mechanisms (dangerouslySetInnerHTML, knex.raw() template literals) and framing them as "ZERO-TOLERANCE" forces deterministic compliance on the exact vectors that lead to XSS and SQLi vulnerabilities.
The Anti-TypeScript Anchor: Because roughly 95% of modern React/Node tutorials in an LLM's training data use TypeScript, the AI is incredibly biased toward it. This prompt attacks that bias in three separate places (Tech Stack Mandate, ES Modules Rule, Negative Constraints) to ensure pure ESM Vanilla JS output.
